# 调试记录

## 问题：小车不动，final_l / final_r 恒为 0

**现象**
- IMU 正常运行，`imu_sys.pitch` 有数据
- 调试器中 `balance_pwm_out` 和 `target_gyro_rate` 值正常（非零）
- 但 `final_l` 和 `final_r` 均为 0，电机无输出

**根因**

`Motor.c` 中声明了转向环 PID 实例：

```c
static PIDInstance pid_turn;
```

但 `Motor_PID_Init()` 里从未调用 `PIDInit(&pid_turn, &config)` 进行初始化，导致：

```
pid_turn.Kp = 0, pid_turn.Ki = 0, pid_turn.Kd = 0
```

虽然 `turn_out = 0` 理论上不影响 `final_l = -(int16_t)(balance_pwm_out + 0)`，  
但实际测试证明：**补上 `pid_turn` 的初始化后小车恢复正常运动**。

> 推测：未初始化的 `PIDInstance` 内部某字段（如 `MaxOut = 0`）导致 `PIDCalculate` 对 `pid_gyro` 的输出也被裁切为 0。

**修复**

在 `Motor_PID_Init()` 末尾补充转向环初始化（`project/code/Moudle/Motor/Motor.c`，约第 500 行）：

```c
// --- 4. 转向环 (Turn Loop) ---
config.Kp = TURN_KP;          // 1.0f
config.Ki = TURN_KI;          // 0.0f
config.Kd = TURN_KD;          // 0.05f
config.dt = CONTROL_DT;       // 1ms
config.MaxOut = MAX_TURN_SPEED; // 2000.0f
config.IntegralLimit = 500.0f;
config.DeadBand = 0.0f;
config.Improve = PID_Integral_Limit;
config.Output_LPF_RC = 0.0f;
config.Derivative_LPF_RC = 0.0f;
PIDInit(&pid_turn, &config);
```

**结论**

所有 `static PIDInstance` 在使用前必须通过 `PIDInit()` 完整初始化，  
仅靠 `Motor_Reset_State()` 清零状态字段不够，`MaxOut` 等配置字段仍为 0，会导致输出被裁切。

## 同期改动：菜单启动预调宏开关

**功能**：通过宏定义控制上电是否进入 PID 预调阶段。

**开关位置**：`project/code/APP/Menu.h`

```c
// 注释掉此行可跳过上电预调，直接进入主菜单开始运行
// #define MENU_STARTUP_PID_TUNE
```

| 状态 | 行为 |
|------|------|
| 定义宏 | 上电进入 "Step 1/4 PRE-TUNE PID"，逐一确认4个参数后启动 |
| 注释宏（默认）| 上电直接进主菜单，`Menu_IsStartupDone()` 立即返回 1 |

**涉及文件**
- `project/code/APP/Menu.h` — 宏定义
- `project/code/APP/Menu.c` — 静态变量初始化、`Menu_Init()`、`STATE_STARTUP_PID` case 三处均加 `#ifdef` 保护
- `project/user/main_cm7_0.c` — 预调等待循环用 `#ifdef MENU_STARTUP_PID_TUNE` 包裹
