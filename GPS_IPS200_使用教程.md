# GPS与IPS200显示屏使用教程

## 1. 概述

本教程介绍如何在CYT4BB开源库项目中使用GPS模块和IPS200显示屏，实现GPS数据的接收、解析和显示功能。

### 1.1 硬件组件

- **GPS模块**：TAU1201 GNSS模块
- **显示屏**：IPS200彩色液晶屏（使用并口8080模式）
- **主控芯片**：CYT4BB（英飞凌TriCore系列）

### 1.2 软件架构

```
应用层
├── GPS模块 (GPS.h/GPS.c)
│   └── zf_device_gnss底层驱动
└── IPS200显示模块 (IPS200.h/IPS200.c)
    └── zf_device_ips200底层驱动
```

## 2. 硬件连接

### 2.1 GPS模块连接

| GPS模块引脚 | CYT4BB引脚 | 说明 |
|------------|-----------|------|
| TX | UART2_RX | GPS数据输出 |
| RX | UART2_TX | GPS数据输入 |
| VCC | 3.3V | 电源正极 |
| GND | GND | 电源地 |

**注意**：UART2配置为115200波特率，10Hz更新频率。

### 2.2 IPS200显示屏连接

IPS200使用并口8080模式连接，具体引脚配置请参考硬件原理图。

## 3. 软件配置

### 3.1 头文件包含

```c
#include "GPS.h"
#include "IPS200.h"
```

### 3.2 初始化顺序

```c
// 1. 系统时钟初始化
clock_init(SYSTEM_CLOCK_250M);

// 2. 调试串口初始化
debug_init();

// 3. GPS模块初始化
GPS_Init();

// 4. IPS200显示屏初始化
IPS200_Init();

// 5. 延时等待模块稳定
system_delay_ms(500);
```

## 4. API函数说明

### 4.1 GPS模块API

#### `GPS_Init()`
**功能**：初始化GPS模块

**参数**：无

**返回值**：无

**说明**：
- 初始化TAU1201 GNSS模块
- 配置UART2串口（115200bps, 10Hz）
- 底层调用`gnss_init(TAU1201)`

#### `GPS_Update()`
**功能**：检查并更新GPS数据

**参数**：无

**返回值**：
- `1`：有新数据已解析
- `0`：无新数据

**说明**：
- 在主循环中轮询调用
- 检查`gnss_flag`标志位
- 有新数据时调用`gnss_data_parse()`解析
- 解析后的数据存储在全局结构体`gnss`中

### 4.2 IPS200显示模块API

#### `IPS200_Init()`
**功能**：初始化IPS200显示屏

**参数**：无

**返回值**：无

**说明**：
- 使用并口8080模式（IPS200_TYPE_PARALLEL8）
- 底层调用`ips200_init(IPS200_MODULE_TYPE)`

#### `IPS200_ShowGNSS()`
**功能**：显示GPS数据

**参数**：无

**返回值**：无

**说明**：
- 显示`gnss`结构体中的数据
- 显示布局（行高16像素）：
  - 第0行：年 月 日
  - 第1行：时 分 秒
  - 第2行：定位状态 纬度
  - 第3行：经度 速度
  - 第4行：航向角 卫星数
  - 第5行：海拔高度

## 5. 使用示例

### 5.1 基本使用流程

```c
#include "zf_common_headfile.h"
#include "GPS.h"
#include "IPS200.h"

int main(void)
{
    // 系统初始化
    clock_init(SYSTEM_CLOCK_250M);
    debug_init();
    system_delay_init();
    
    // GPS和显示屏初始化
    GPS_Init();
    IPS200_Init();
    system_delay_ms(500);
    
    printf("GPS and IPS200 initialized.\r\n");
    
    // 主循环
    while(true)
    {
        // 检查GPS数据更新
        if(GPS_Update())
        {
            // 有新数据时刷新显示
            IPS200_ShowGNSS();
            
            // 可选：通过串口输出数据
            printf("Lat: %.6f, Lon: %.6f\r\n", 
                   gnss.latitude, gnss.longitude);
        }
        
        // 可选：添加延时降低刷新率
        // system_delay_ms(100);
    }
}
```

### 5.2 数据结构说明

GPS数据存储在全局结构体`gnss`中，主要字段包括：

```c
// 时间信息
gnss.time.year    // 年
gnss.time.month   // 月
gnss.time.day     // 日
gnss.time.hour    // 时
gnss.time.minute  // 分
gnss.time.second  // 秒

// 位置信息
gnss.latitude     // 纬度（度）
gnss.longitude    // 经度（度）
gnss.height       // 海拔高度（米）

// 速度和方向
gnss.speed        // 速度（公里/小时）
gnss.direction    // 航向角（度）

// 状态信息
gnss.state        // 定位状态
gnss.satellite_used // 使用的卫星数
```

## 6. 注意事项

### 6.1 初始化顺序
- 必须先初始化GPS，再初始化显示屏
- 建议添加适当的延时等待模块稳定

### 6.2 数据更新频率
- GPS模块配置为10Hz更新频率
- 主循环轮询间隔应小于100ms以确保不丢失数据
- 可根据需要调整轮询频率

### 6.3 显示刷新
- 仅在GPS有新数据时刷新显示，避免频繁刷新导致屏幕闪烁
- 显示内容使用固定字体大小，布局已优化

### 6.4 调试信息
- 可通过调试串口输出GPS数据用于调试
- 注意串口波特率设置（默认115200）

### 6.5 电源管理
- GPS模块需要稳定的3.3V电源
- 确保电源纹波小于50mV以获得最佳性能

## 7. 常见问题

### 7.1 GPS无数据
**现象**：GPS_Update()始终返回0

**检查步骤**：
1. 检查UART2连接是否正确
2. 确认GPS模块电源正常
3. 检查天线连接
4. 确认模块在室外开阔环境
5. 查看调试串口是否有错误信息

### 7.2 显示异常
**现象**：屏幕无显示或显示乱码

**检查步骤**：
1. 确认IPS200连接正确
2. 检查并口8080模式配置
3. 确认电源电压稳定
4. 检查初始化顺序

### 7.3 数据不更新
**现象**：GPS数据长时间不变化

**检查步骤**：
1. 确认GPS模块已定位（gnss.state显示定位状态）
2. 检查卫星数量（至少需要4颗卫星）
3. 确认模块在室外环境
4. 检查模块是否损坏
